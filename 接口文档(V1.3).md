# 智慧校园生活助手——接口文档 V1.3

## 一、概述

### 1.1 基本信息

- **API Base URL**: `http://api.example.com/v1`（实际部署时替换）
- **数据格式**: JSON
- **字符编码**: UTF-8
- **认证方式**: JWT Bearer Token

### 1.2 通用请求头

| Header | 类型 | 必填 | 说明 |
|--------|------|------|------|
| Content-Type | string | 是 | application/json |
| Authorization | string | 是 | Bearer {token} |

### 1.3 通用响应格式

```json
{
  "code": 200,
  "message": "操作成功",
  "data": {}
}
```

### 1.4 错误码规范

| code | 说明 |
|------|------|
| 200 | 成功 |
| 400 | 请求参数错误 |
| 401 | 未授权 / Token 无效 |
| 403 | 权限不足 / 账号冻结 / 需修改临时密码（见下表 subCode） |
| 404 | 资源不存在 |
| 409 | 业务冲突 |
| 500 | 服务器内部错误 |

**403 错误子类型（subCode）**

| code | subCode | 说明 |
|------|---------|------|
| 403 | insufficient_permission | 权限不足，无法执行此操作 |
| 403 | account_frozen | 账号已被冻结，请联系管理员 |
| 403 | temp_password_required | 请先修改临时密码 |

### 1.5 通用规范说明

#### 分页参数规范

| 参数名 | 类型 | 必填 | 默认值 | 说明 |
|--------|------|------|--------|------|
| page | integer | 否 | 1 | 页码 |
| pageSize | integer | 否 | 20 | 每页数量（最大 100） |

#### 分页响应格式

```json
{
  "code": 200,
  "message": "获取成功",
  "data": {
    "total": 100,
    "page": 1,
    "pageSize": 20,
    "list": []
  }
}
```

#### 数据脱敏规范

手机号返回时中间 4 位脱敏，例如：`138****8000`

#### Token 过期处理

Token 过期返回 401，前端应调用 `POST /auth/refresh` 刷新 Token。

### 1.6 违规检测机制

系统自动检测以下行为并记录异常日志：

| 违规类型 | 检测范围 | 处置方式 | 涉及接口 |
|----------|----------|----------|----------|
| 内容违规 | 昵称、日程备注、课程名称、公告内容 | 拒绝提交 + 记录异常日志 | `PUT /user/profile`、`POST /schedule/events`、`PUT /schedule/events/{eventId}`、`POST /course/import`、`POST /course`、`PUT /course/{courseId}`、`POST /admin/announcements` |
| 异常打卡 | 睡眠/运动/体重录入（1 分钟内提交次数超过阈值） | 立即冻结账号 + Token 失效 + 同时写入操作日志和异常日志 | `POST /health/sleep`、`POST /health/exercise`、`POST /health/weight` |

管理员可通过 `GET /admin/logs/exception` 查询异常日志，通过 `GET /admin/logs/operation?action=auto_freeze_user` 查询自动冻结记录。

### 2.2 用户登录

**请求信息**

| 项目 | 说明 |
|------|------|
| URL | POST /auth/login |
| 认证 | 否 |

**说明**：系统采用统一登录入口，用户和管理员均使用此接口登录。登录成功后根据返回的 `role` 字段判断跳转：`user` → 学生端，`admin` / `super_admin` → 管理后台。

**请求参数**

| 参数名 | 类型 | 必填 | 说明 |
|--------|------|------|------|
| phone | string | 是 | 手机号 |
| password | string | 是 | 登录密码 |

**响应示例**

```json
{
  "code": 200,
  "message": "登录成功",
  "data": {
    "userId": "usr_xxxxxxxx",
    "nickname": "用户_A1b2C3d4",
    "avatar": "https://cdn.example.com/avatar/default.png",
    "token": "eyJhbGciOiJIUzI1NiIsInR5cCI6IkpXVCJ9...",
    "refreshToken": "eyJhbGciOiJIUzI1NiIsInR5cCI6IkpXVCJ9...",
    "expiresIn": 604800,
    "role": "user"
  }
}
```

**临时密码登录响应**

当用户使用管理员重置的临时密码登录时，响应中增加 `forceChangePassword: true`，前端应引导用户调用 `POST /user/force_password` 修改密码：

```json
{
  "code": 200,
  "message": "登录成功，请先修改临时密码",
  "data": {
    "userId": "usr_xxxxxxxx",
    "token": "eyJhbGciOiJIUzI1NiIsInR5cCI6IkpXVCJ9...",
    "refreshToken": "eyJhbGciOiJIUzI1NiIsInR5cCI6IkpXVCJ9...",
    "expiresIn": 604800,
    "role": "user",
    "forceChangePassword": true
  }
}
```

**响应 role 字段说明**

| 值 | 说明 |
|----|------|
| user | 普通用户，跳转至学生端 |
| admin | 普通管理员，跳转至管理后台 |
| super_admin | 超级管理员，跳转至管理后台 |

**冻结账号登录响应**

```json
{
  "code": 403,
  "subCode": "account_frozen",
  "message": "您的账号因[内容违规]已被冻结，请联系管理员"
}
```

### 2.7 更新用户信息

**请求信息**

| 项目 | 说明 |
|------|------|
| URL | PUT /user/profile |
| 认证 | 是 |

**说明**：修改体重时，系统自动在 `weight_record` 表生成当天打卡记录。

**安全说明**：昵称字段将进行内容违规检测，若包含违禁词汇将拒绝保存并记录异常日志。

**请求参数**

| 参数名 | 类型 | 必填 | 说明 |
|--------|------|------|------|
| nickname | string | 否 | 昵称，2-15 字符，支持中英文、数字、下划线 |
| gender | string | 否 | 性别：未知 / 男 / 女 |

| birthDate | string | 否 | 出生日期，格式：YYYY-MM-DD |
| department | string | 否 | 院系/专业 |
| height | integer | 否 | 身高，100-250 cm |
| weight | decimal | 否 | 体重，30-250 kg，保留一位小数 |

**请求示例**

```json
{
  "nickname": "学霸小明",
  "gender": "男",
  "grade": "大二",
  "birthDate": "2004-06-15",
  "department": "计算机学院",
```

1. 冻结用户账号

3. 在 `exception_log` 表中记录异常日志

管理员可通过以下接口查询：

- `GET /admin/logs/operation?action=auto_freeze_user` 查询所有自动冻结操作记录

## 版本历史

### V1.3

1. **POST /auth/send_verify_code**
   - **变更内容**：正文路径由 `/auth/send_verifyCode` 统一修正为 `/auth/send_verify_code`；`type` 参数枚举增加 `admin_create`。
   - **新增/改动功能**：与接口路由命名规范一致；支持创建管理员场景的验证码发送。

2. **POST /auth/login**
   - **变更内容**：响应示例增加 `refreshToken`、`expiresIn` 字段；新增临时密码登录响应示例（含 `forceChangePassword: true`）；冻结响应增加 `subCode: account_frozen`。
   - **新增/改动功能**：完善双 Token 认证流程；支持临时密码登录后引导强制改密。

3. **POST /auth/refresh**
   - **变更内容**：认证改为「否」；请求参数改为 `refreshToken`；响应返回新的 `token`、`refreshToken`、`expiresIn`。
   - **新增/改动功能**：使用 refreshToken 刷新访问令牌，无需携带已过期的 access token。

4. **POST /user/force_password**（新增）
   - **变更内容**：V1.2 缺失，V1.3 新增完整接口定义。
   - **新增/改动功能**：临时密码登录后强制修改密码；未修改时其他接口返回 `subCode: temp_password_required`。

5. **POST /admin/users/{userId}/reset_password**
   - **变更内容**：新增权限边界说明表格。
   - **新增/改动功能**：明确普通管理员仅可重置 user、超级管理员可重置 user/admin、超级管理员密码不可被重置。

6. **POST /admin/admins**
   - **变更内容**：`verifyCode` 说明明确验证码类型须为 `admin_create`。
   - **新增/改动功能**：与验证码类型枚举联动。

7. **POST /schedule/events**
   - **变更内容**：`category` 枚举恢复 `course`；移除禁止创建课程的 403 说明。
   - **新增/改动功能**：支持手动添加课程类日程，同时创建 `course` 与 `schedule_event` 记录。

8. **GET /course/list**（新增）
   - **变更内容**：V1.2 缺失，V1.3 新增完整接口定义。
   - **新增/改动功能**：获取当前学期课程列表。
